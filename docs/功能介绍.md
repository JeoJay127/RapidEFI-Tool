# RapidEFI 功能介绍

这份文档面向第一次接触 RapidEFI 的用户，重点说明几个常用入口分别适合什么情况、能解决什么问题，以及使用时需要注意什么。

## 1. 手动配置 EFI

**适合谁：** 已经了解 OpenCore 基本概念，知道自己的 CPU 平台、显卡、声卡、网卡和 macOS 版本需求的用户。

**能做什么：**

- 手动选择 CPU 类型、平台类型、平台信息和 macOS 版本。
- 配置 SMBIOS、核显、独显、声卡、USB、网卡、WiFi、蓝牙、触摸板等常见项目。
- 勾选可选 Kext、启动参数、UEFI 驱动、OpenCore Quirks 等配置。
- 输出完整 EFI，并保存 `configModel`，方便后续继续加工。

**什么时候用：**

- 你已经知道自己的硬件平台，例如 Intel Desktop 4 代、AMD Ryzen Desktop、Intel Laptop 等。
- 你希望自己控制每一项驱动和参数。
- 自动配置结果需要进一步手动调整。

**注意事项：**

- 手动配置更灵活，但也更依赖经验。
- 不确定的选项不要随意勾选，尤其是显卡、ACPI、启动参数和 SIP/OCLP 相关设置。
- 输出 EFI 后建议先用 U 盘测试启动。

## 2. 硬件信息查看与 macOS 兼容性提示

**适合谁：** 不确定自己硬件能不能安装 macOS，或者想让工具根据硬件信息自动配置 EFI 的用户。

**能做什么：**

- 查看 CPU、主板、显卡、声卡、网卡、WiFi、蓝牙、磁盘、USB 控制器等硬件信息。
- 显示硬件在 macOS 下的大致兼容性提示。
- 导出本机硬件报告和 ACPI 表，方便给别人分析或在其他电脑上导入。
- 导入其他电脑的硬件报告和 ACPI 表，用于为另一台机器制作 EFI。

**什么时候用：**

- 第一次制作 EFI，不清楚硬件细节。
- 想知道显卡、WiFi、网卡、磁盘是否可能需要额外驱动或补丁。
- 需要把目标电脑的硬件资料带到另一台电脑上制作 EFI。

**注意事项：**

- 兼容性提示是辅助判断，不代表最终一定能启动或免调试。
- 导入其他电脑硬件资料时，如果想定制 SSDT，需要同时提供该电脑的 ACPI 表目录。
- 如果没有导入 ACPI 表，外部硬件资料通常只能使用预制或原始 SSDT 方案。

## 3. 自动配置 EFI

**适合谁：** 新手用户，或者想先快速生成一个基础 EFI 再慢慢调整的用户。

**能做什么：**

- 根据硬件信息自动推断 CPU 平台和机型方向。
- 自动选择常用驱动、启动参数、设备属性和部分修复项。
- 根据硬件兼容性处理部分显卡、WiFi、网卡、磁盘和控制器需求。
- 在支持的环境下，根据 ACPI 表自动定制 SSDT。
- 对部分不支持的硬件设备进行 SSDT 屏蔽。
- 对需要仿冒的显卡生成对应的 SSDT 仿冒补丁。

**什么时候用：**

- 你不知道应该手动选择哪些配置。
- 你希望工具根据硬件报告先生成一份基础 EFI。
- 你想减少从零配置 OpenCore 的时间。

**注意事项：**

- 自动配置不是万能方案，生成结果仍建议检查。
- 不同主板 BIOS、显卡连接方式、无线网卡型号都会影响最终结果。
- 如果自动配置无法启动，可以使用加工 EFI 或手动配置继续调整。
- 定制 SSDT 依赖 ACPI 表，给其他电脑制作 EFI 时请尽量导入该电脑的 ACPI 表目录。

## 4. 加工 EFI

**适合谁：** 已经用 RapidEFI 生成过 EFI，希望继续调整而不想手动编辑 `config.plist` 的用户。

**能做什么：**

- 导入 RapidEFI 输出目录中的 `configModel`。
- 再次进入熟悉的配置页面调整 EFI。
- 去掉 `-v` 跑码、修改启动参数、调整 Kext、切换显卡或声卡设置。
- 重新输出调整后的 EFI。

**什么时候用：**

- EFI 已经能启动，但想去掉跑码或优化参数。
- 安装完成后想把调试配置改成日常使用配置。
- 需要对比不同驱动组合或不同启动参数。

**注意事项：**

- 加工 EFI 依赖 RapidEFI 生成的 `configModel` 文件。
- 如果手动改过 `config.plist`，但没有同步更新 `configModel`，加工结果可能不会包含你手动改动的内容。
- 建议保留旧 EFI 备份，再输出新的 EFI。

## 5. 历史记录

**适合谁：** 经常制作或调整 EFI，需要回看旧配置、重新导出或对比不同版本的用户。

**能做什么：**

- 自动保存 RapidEFI 成功生成 EFI 的历史记录。
- 查看以前制作过的 EFI 配置。
- 基于历史记录再次编辑或重新导出。
- 帮助对比不同配置之间的变化。

**什么时候用：**

- 某次 EFI 可以启动，后续调整后反而出问题，需要回到旧版本。
- 给多台机器制作 EFI，需要查找之前的配置。
- 想保留不同 macOS 版本或不同硬件组合的配置记录。

**注意事项：**

- 历史记录是为了方便回看和二次编辑，不等同于完整系统备份。
- 清理历史记录前，请确认不再需要对应配置。
- 重要 EFI 建议额外复制到安全位置保存。

## 推荐使用方式

如果你是新手，可以按这个顺序尝试：

1. 先进入 **自动配置 EFI**，刷新或导入硬件信息。
2. 查看硬件兼容性提示，确认是否有明显不支持的设备。
3. 在 **EFI 设置** 中确认 macOS 版本、声卡布局、SSDT 方案等。
4. 输出 EFI，并使用 U 盘测试启动。
5. 能启动后，再通过 **加工 EFI** 去掉调试参数或继续细调。
6. 如果某次配置效果更好，可以通过 **历史记录** 找回并再次导出。

RapidEFI 的定位不是替你省掉所有判断，而是把复杂步骤整理成更容易理解和继续调整的流程。
